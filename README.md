# Deck Masters AK

Static marketing site for a cold-climate deck builder and general contractor. **Astro 5**, **Tailwind CSS**, **Cloudflare Pages** (static + Functions).

**Performance-first:** The site is built for high Lighthouse scores—zero client-side JS where possible, server-rendered HTML, optimized images (including `astro:assets` for the Experimental reviews avatars), and minimal third-party requests.

## Google Reviews (DataForSEO)

Reviews can be synced from the **DataForSEO Business Data API** (search by business name). This powers the **Experimental** reviews section on the homepage and keeps avatars local for performance.

**Flow:**

1. **Config** — `scripts/reviews-clients.json` defines clients (keyword, location, language). Add one entry per business.
2. **Credentials** — `.env` in project root: `DATAFORSEO_LOGIN`, `DATAFORSEO_PASSWORD` (see `.env.example`).
3. **Sync** — Run `npm run sync-reviews` (default client) or `node --env-file=.env scripts/sync-reviews.js <slug>` for a specific client.
4. The script creates a task via `task_post`, polls until ready, downloads profile avatars to `src/assets/avatars/`, maps to the existing `rawReviews` schema, and writes `src/data/google-reviews.json`.
5. **Component** — `ReviewsExperimental.astro` reads that JSON and uses `astro:assets` (glob + `getImage`) to optimize avatars at build time. No client-side JS.

Full details (adding clients, location codes, output paths): **`scripts/README.md`**.

## Project structure

- `src/data/site.json` — Business info, services, neighborhoods.
- `src/data/reviews.json` — Primary reviews (e.g. Stoke/MoreGood).
- `src/data/google-reviews.json` — DataForSEO-synced reviews (generated by `sync-reviews`).
- `src/pages/` — index, services, about, contact.
- `src/components/` — Hero, TrustBar, GoogleReviewCard, ReviewsExperimental, QuoteForm, etc.
- `scripts/sync-reviews.js` — DataForSEO Google Reviews sync (task_post → task_get → avatars → JSON).
- `scripts/reviews-clients.json` — Client config (keyword, location, output paths).
- `functions/api/submit-quote.js` — POST /api/submit-quote; creates a GHL contact with tags.

`public/newhero.avif` and `public/newhero.png` are loaded **only on desktop** (via `<picture>` + `media="(min-width: 1024px)"` in `Hero.astro`); mobile uses a gradient so LCP stays fast. Prefer AVIF as primary; PNG as fallback.

## Deploy (Cloudflare Pages)

Connect the repo in [Cloudflare Dashboard](https://dash.cloudflare.com) → Workers & Pages → Pages → Connect to Git.

- **Build command:** `npm run build`
- **Build output directory:** `dist`

Cloudflare builds and deploys automatically on push. Ensure `wrangler.toml` has `name` and `pages_build_output_dir = "dist"` so the `functions/` API is used.

## Quote form & GoHighLevel

Form submits to **POST /api/submit-quote**. The Function creates a contact in GHL: name, email, phone, source "Website Quote Form", and **tags** (service type, neighborhood, `quote`, `website`). Project description can go in a GHL custom field if you set the env var.

**Env vars** (Cloudflare Pages → Settings → Environment variables):

- `HIGHLEVEL_TOKEN` — required (scopes: **contacts.write**; **locations/customFields.write** only if using project description custom field)
- `HIGHLEVEL_LOCATION_ID` — required
- `HIGHLEVEL_CUSTOM_FIELD_PROJECT_DESCRIPTION` — optional (GHL custom field ID for project description)

**Troubleshooting:** Form only works on the deployed site (not local dev). If the form fails, check the latest deployment is Live, env vars are set, and hard-refresh or use incognito. "Server configuration error" = missing env vars. Build error "Return statement is not allowed" = use an `if (condition) { ... }` block in Astro script instead of top-level `return`. Deploy "Must specify a project name" = `wrangler.toml` needs `name` and `pages_build_output_dir = "dist"`.
